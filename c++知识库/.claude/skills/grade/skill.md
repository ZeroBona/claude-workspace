# Skill: grade

批改试卷、生成成绩、整理错题。

**目标用户**：初一学生（零基础，12~13岁）
**语言要求**：评语亲切鼓励，错题解析要通俗易懂

## 作答方式

学生在 `05_学习系统/01_试题库/` 下的试题文件**直接填写答案**：

- 选择题 → 在 `（ ）` 中填选项字母，如 `（ B ）`
- 判断题 → 在 `（ ）` 中填 `✅` 或 `❌`
- 填空题 → 替换 `____` 为填写内容
- 读程序题 → 在 `______` 处写运行结果
- 编程题 → 在题后写代码

没做的题留空即可。

## 输入方式（自然语言）

| 用户说 | 行为 |
|--------|------|
| `批改` | 询问要批改哪个单元 |
| `批改01` | 读取试题文件中的作答，批改第01单元 |
| `批改01 小明` | 批改第01单元，注明学生姓名小明 |

## 工作流程

### 1. 确定批改目标

从用户输入中提取单元编号，映射到对应的试题文件 `05_学习系统/01_试题库/XX_单元名称.md`。

### 2. 检查试题是否存在

**如果不存在**：提示"该单元还没有试题，请先出题"

### 3. 读取学生作答

直接读取试题文件，从以下位置提取学生填写的答案：

| 题型 | 提取位置 | 示例 |
|------|---------|------|
| 选择题 | `（ ）` 括号内 | `（ B ）` → 答案 B |
| 判断题 | `（ ）` 括号内，1=对 0=错 | `（ 1 ）` → 对，`（ 0 ）` → 错 |
| 填空题 | `____` 被替换的内容 | `cout` → 答案 cout |
| 读程序题 | `______` 处写的内容 | 运行结果文本 |
| 编程题 | 题后的代码 | 学生写的代码 |

**判断是否作答**：括号内空白或 `____` 未被替换 → 视为未做。

### 4. 生成标准答案（内部对照）

根据知识库内容现场生成标准答案，不写入文件。

### 5. 逐题批改

逐题比对，记录答题情况：

| 题号 | 标准答案 | 学生答案 | 结果 | 知识点 |
|------|---------|---------|------|--------|
| 1 | B | B | ✅ | 程序入口 |
| 2 | C | A | ❌ | cout用法 |
| 3 | B | （空） | - | cin用法 |

### 6. 计算得分

**计分规则**：没填的空不算分，不计入总分基数。

- **未做** → 该题分值从总分中扣除
- **正确** → 得全部分值
- **错误** → 得 0 分

得分公式：
```
每题分值 = 该题型总分 ÷ 题量
实得分 = 答对题数 × 每题分值
总分基数 = 已做题数 × 每题分值
```

| 题型 | 题量 | 每题分值 | 答对 | 未做 | 得分 |
|------|------|---------|------|------|------|
| 选择题 | 10 | 3 | 7 | 1 | 21/27 |
| 判断题 | 10 | 2 | 8 | 2 | 16/16 |
| 填空题 | 6 | 3 | 4 | 0 | 12/18 |
| 读程序 | 3 | 4 | 2 | 1 | 8/8 |
| 编程题 | 2 | 10/10 | - | - | 酌情给分 |

### 7. 记录错题 → 错题库

将答错的题整理到 `05_学习系统/02_错题库/XX_单元名称.md`：

```markdown
### 错题 N
- **学生**：小明
- **日期**：2026-06-XX
- **题型**：选择题
- **题目**：（题目原文）
- **我的答案**：A
- **正确答案**：C
- **知识点**：xxx
- **订正**：（解析）
```

### 9. 等级标准

| 得分率 | 等级 |
|--------|------|
| 90%~100% | A（优秀 🌟） |
| 80%~89% | B（良好 👍） |
| 60%~79% | C（及格 💪） |
| 0%~59% | D（需努力 📖） |
