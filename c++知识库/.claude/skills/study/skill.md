# Skill: study

展示 C++ 课程的整体概览或指定单元的学习内容。

**目标用户**：初一学生（零基础，12~13岁）
**语言要求**：通俗易懂，语气亲切有鼓励性，像学长/学姐在介绍课程

## 输入方式（自然语言）

用户在对话中直接说中文即可调用：

| 用户说 | 行为 |
|--------|------|
| `学习` 或 `学习`（不带数字） | 展示整个 C++ 课程的**全景概览**——14个单元的总览图、学习路线、当前进度 |
| `学习01` 或 `学习 01` | 展示**第01单元**的详细内容介绍 |
| `学习 05` | 展示第05单元的详细内容介绍 |

## 工作流程

### 1. 解析目标单元编号

从用户的输入中提取数字：
- `"学习01"` → 提取 `01`
- `"学习 03"` → 提取 `03`
- `"学习"`（无数字）→ 展示全课程总览

单元编号映射：
```
01 → 01_编程初识与C++入门
02 → 02_变量与数据类型
...
14 → 14_综合实战
```

### 2. 无参数 → 展示课程全景总览

读取以下信息，组合成课程全景报告：

#### 2.1 课程基本信息
- 课程名称：C++ 编程入门
- 总单元数：14 个
- 目标学生：初一零基础

#### 2.2 各单元概要

从 `01_核心知识库/` 下每个单元的 `README.md` 中读取：
- 单元标题和 emoji
- 学习目标（list）
- 预计学习时间
- 前置要求

整理成表格或卡片形式展示。

#### 2.3 学习进度

从 `05_学习系统/01_试题库/README.md` 读取试题状态，判断每个单元的学习进度：

| 状态 | 含义 |
|------|------|
| ⏳ 待出 | 还没学到这 |
| ✅ 已出 | 有试题可做 |
| （有错题记录） | 已批改过，有错题可复习 |

从 `05_学习系统/02_错题库/` 读取各单元错题数，标记薄弱环节。

#### 2.4 建议下一步

根据当前进度给出学习建议，例如：
- 如果所有单元都还没开始 → "从第01单元开始吧！"
- 如果有单元已出题但尚未批改 → "做完题记得说'批改'哦！"
- 如果有错题 → "有 X 道错题，说'复习'巩固一下吧！"

### 3. 带参数 → 展示指定单元详情

读取指定单元的 `README.md`，展示：

#### 3.1 单元概览卡片

```
╔══════════════════════════════╗
║  第01单元：编程初识与C++入门 🚀 ║
║  ──────────────────────────  ║
║  ⏱ 预计学习时间：60~90分钟     ║
║  📋 前置要求：不需要任何基础！   ║
╚══════════════════════════════╝
```

#### 3.2 学习目标（可视化）

每个目标前面显示完成状态：
- 从错题库/试题库判断是否已有学习痕迹
- 如果是新单元，所有目标显示为"待学习"

#### 3.3 学习路线导航

从 `README.md` 的学习导航部分展示学习步骤。

#### 3.4 当前进度

- 该单元试题状态（未出/已出/已批改）
- 该单元错题数（如果有）
- 鼓励性建议

### 5. 结尾祝福语

不论哪种情况，输出结尾都要附带一句祝福语，例如：
- "🌈 学习愉快，加油！🎉"
- "📚 每天进步一点点，坚持就是胜利！💪"
