# Skill: exam

按照核心知识库学习进度生成标准化试题。

**目标用户**：初一学生（零基础，12~13岁）
**语言要求**：通俗易懂，多用生活类比，语气亲切鼓励

## 输入方式（自然语言）

用户在对话中直接说中文即可调用，skill 会自动解析：

| 用户说 | 行为 |
|--------|------|
| `出题` 或 `出题`（不带数字） | 扫描所有单元，找**第一个没有试题的单元**出题 |
| `出题01` 或 `出题 01` | 出**指定单元**的题 |
| `出题 02` | 出第02单元的题 |

## 工作流程

### 1. 解析目标单元编号

从用户的输入中提取数字：
- `"出题01"` → 提取 `01`
- `"出题 03"` → 提取 `03`
- `"出题"`（无数字）→ 自动找下一个

找到单元编号后，映射到对应的文件夹名：
```
01 → 01_编程初识与C++入门
02 → 02_变量与数据类型
...
14 → 14_综合实战
```

### 2. 检查是否已存在

检查 `05_学习系统/01_试题库/` 下是否有该单元的试题文件。

**如果已存在**：输出提示信息，询问用户是否要重新出题覆盖，或跳过。

**如果不存在**：继续出题流程。

### 3. 自动定位（无参数时）

列出 `01_核心知识库/` 下所有单元，对照 `05_学习系统/01_试题库/` 中已有的试题文件，找出**第一个未出题的单元**作为目标。

如果所有单元都已出题，提示"🎉 所有单元都已出题完毕！"

### 4. 阅读单元内容

读取目标单元的：
- `README.md` — 学习目标和知识点
- `01_讲解.md` — 核心知识点
- `02_代码示例.md` — 代码示例
- `03_练习题.md` — 已有练习题

### 5. 生成标准化试题

在 `05_学习系统/01_试题库/` 下创建 `XX_单元名称.md` 文件。

#### 试卷结构（满分100分）

| 题型 | 题量 | 分值 | 说明 |
|------|------|------|------|
| 一、选择题 | 10题 | 30分 | 基础概念，每题3分 |
| 二、判断题 | 10题 | 20分 | 易错辨析，每题2分 |
| 三、填空题 | 5~6题 | 18~20分 | 代码补全/概念填空 |
| 四、读程序写结果 | 3题 | 12~15分 | 分析运行输出 |
| 五、编程题 | 2题 | 20分 | 实际编码 |

#### 出题原则

1. **难度分层**：基础题约60%，进阶题约30%，挑战题约10%
2. **语言适配**：面向初一学生，题干表述清晰易懂
3. **覆盖全面**：涵盖该单元所有核心知识点
4. **参考答案不出现在试题文件中**：答案只在批改时内部生成对照，试题文件里不写参考答案
5. **代码可运行**：读程序题和编程题的代码必须能通过编译

#### 题型格式规范

**选择题格式**：
```
**1.** （  ）编程的本质是什么？
A. 选项一
B. 选项二
C. 选项三
D. 选项四
```
- 题号后加 `（  ）` 让学生填选项字母
- 每个选项单独占一行
- 题干后面空行再接选项

**判断题格式**：
```
**11.** （  ）编程就是写指令。
```
- 题号后加 `（  ）` 让学生填 1（对）或 0（错）

**填空题格式**：
```
**21.** 完整的程序框架使用 `______` 关键字。
```
- 填空用 `______`（6个下划线）
- 不需要填多个空的，每个空独立

**读程序题格式**：
```
**26.** 阅读以下代码，写出运行结果：
```cpp
...
```
运行结果：
```
______
```
```
- 结果区用 `______` 让学生填写

**编程题格式**：
- 正常写题目要求，无需预留答题区，学生在 Dev-C++ 中编写实际代码

### 6. 更新题库索引

更新 `05_学习系统/01_试题库/README.md` 中该单元的状态为「✅ 已出」

### 7. 输出确认

向用户报告：
- 哪个单元出了题
- 试卷总分和题型分布
- 核心考察知识点
