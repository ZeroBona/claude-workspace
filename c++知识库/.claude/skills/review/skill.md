# Skill: review

根据错题库中的记录，生成针对性复习练习题，哪里薄弱练哪里。

**目标用户**：初一学生（零基础，12~13岁）
**语言要求**：题目设计贴合初一认知水平，解析用类比帮助理解

## 输入方式（自然语言）

用户在对话中直接说中文即可调用：

| 用户说 | 行为 |
|--------|------|
| `复习` 或 `复习`（不带数字） | **全面复习**——分析所有错题，按优先级出练习 |
| `复习01` 或 `复习 01` | **单元复习**——只针对指定单元的错题 |

## 工作流程

### 1. 读取错题数据

读取 `05_学习系统/02_错题库/` 下所有单元的错题记录，进行统计分析：

统计维度：
- **按单元**：哪个单元错题最多？
- **按题型**：哪个题型最容易错？
- **按知识点**：哪些知识点反复出错？

### 2. 确定复习范围

- 用户带了数字（如 `复习02`）→ 只分析指定单元的错题
- 用户没带数字（如 `复习`）→ 分析所有错题

如果指定单元没有错题记录，提示："该单元还没有错题记录，真棒！🎉"

如果所有单元都没有错题，提示："错题库为空，太厉害了！继续加油！🎉"

### 3. 生成针对性练习题

根据错题分析结果，为每个薄弱知识点生成**同类型但不同题干的练习题**：

#### 文件命名规则

复习卷保存在 `05_学习系统/03_复习卷/` 下，命名规则：

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 单单元复习 | `复习_01_单元名.md` | `复习_02_变量与数据类型.md` |
| 全面复习 | `复习_综合_日期.md` | `复习_综合_2026-06-09.md` |

试卷格式和试题库一致，学生可以直接在文件中填写答案，然后用 `/grade` 批改。

#### 生成原则

1. **换数不换型**：选择题错的是"变量命名规则"，生成新的命名规则判断题
2. **举一反三**：读程序题错了，换一组数字生成类似的读程序题
3. **由简到繁**：先基础巩固，再进阶加深
4. **每道题附解析**：不仅给答案，还要解释"为什么"

#### 难度策略

| 错题次数 | 推荐策略 |
|---------|---------|
| 1次 | 出 1~2 道同类题巩固 |
| 2次 | 出 2~3 道同类题 + 1道变式题 |
| 3次以上 | 出 3~4 道同类题 + 2道变式题 + 1道综合题 |

### 4. 输出复习卷

格式示例：

```
╔══════════════════════════════╗
║  针对性复习练习               ║
║  基于错题分析                 ║
╚══════════════════════════════╝

📊 错题分析报告
─────────────────────
薄弱单元：02_变量与数据类型（错3题）
薄弱题型：选择题（错2题）
薄弱知识点：
  • 变量命名规则 ⚠️⚠️
  • char vs string 引号区别 ⚠️

📝 针对性练习（共 5 题）
─────────────────────

【知识点：变量命名规则】
第1题（巩固）：
...
→ 答案与解析：...

【知识点：char vs string 引号】
第2题（巩固）：
...
→ 答案与解析：...
```

### 5. 跟踪复习效果（可选）

如果学生做完复习题后进行批改：
- 新错的题追加到对应单元的错题库
- 某知识点反复出错 → 标记为"反复错误 ⚠️⚠️⚠️"，建议回看教材

### 6. 输出复习报告

```
📋 复习总结
─────────────────────
覆盖单元：1 个
覆盖知识点：2 个
生成练习：5 题
预计用时：15 分钟

💡 建议：
- 完成练习后使用 /grade 批改
- 如果还是做错，建议回看 01_讲解.md 对应章节
```
