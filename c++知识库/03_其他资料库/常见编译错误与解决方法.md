# 常见编译错误与解决方法 🐛

> 编译报错看不懂？别慌，这里帮你整理了新手最常见的错误！

---

## 📋 错误信息速查

### 1. `[Error] expected ';' before ...`
**缺少分号**

```cpp
// ❌ 错误
cout << "Hello"

// ✅ 正确
cout << "Hello";
```
> C++ 每条语句结尾必须有 `;`，就像句子结尾要有句号。

---

### 2. `[Error] 'xxx' was not declared in this scope`
**变量未定义 / 拼写错误**

```cpp
// ❌ 错误
int a = 10;
cout << A;      // 变量名大小写写错了

// ✅ 正确
int a = 10;
cout << a;
```
> C++ 区分大小写，`age` 和 `Age` 是两个不同的变量！还要检查变量名是否拼对了。

---

### 3. `[Error] redefinition of 'int main'`
**main 函数重复定义**

```cpp
// ❌ 错误——写了两个 main！
int main() { ... }
int main() { ... }

// ✅ 正确——只能有一个 main
int main() { ... }
```
> 一个程序只能有一个 `main()` 函数。

---

### 4. `[Error] 'string' does not name a type`
**忘记写 `using namespace std;`**

```cpp
// ❌ 错误
#include <iostream>
int main() {
    string name = "小明";  // string 不认识
}

// ✅ 正确
#include <iostream>
using namespace std;       // 告诉编译器用标准命名空间
int main() {
    string name = "小明";
}
```

---

### 5. `[Error] expected primary-expression before ...`
**括号不匹配**

```cpp
// ❌ 错误
cout << (5 + 3) * 2;

// ✅ 正确
cout << (5 + 3) * 2;
```

仔细数数左右括号数量是否相等！
> 小技巧：写左括号 `(` 时马上写右括号 `)`，再在中间填内容。

---

### 6. `[Error] 'else' without a previous 'if'`
**else 前面没有 if**

```cpp
// ❌ 错误
int a = 5;
if (a > 3) cout << "大";
else cout << "小";

// ✅ 正确——else 前必须紧跟着 if 或 else if
if (a > 3) {
    cout << "大";
} else {
    cout << "小";
}
```
> ✅ **好习惯**：if 和 else 后面都加大括号 `{}`，即使只有一行。

---

### 7. `[Error] 'for' loop initial declarations are only allowed in C99 or C11 mode`
**在 for 里定义变量但编译器不识别**

> Dev-C++ 旧版本可能会出现。解决方法：在菜单「工具 → 编译选项」中勾选「编译时加入以下命令」，输入 `-std=c++11`。

```cpp
// 新版 C++ 标准才支持在 for 里定义变量
for (int i = 0; i < 10; i++) { ... }
```

---

### 8. `[Warning] comparison between signed and unsigned integer`
**有符号数和无符号数比较**

```cpp
// 常见于 .size() 的比较
vector<int> v;
for (int i = 0; i < v.size(); i++) { ... }  // ⚠️

// ✅ 改用 size_t
for (size_t i = 0; i < v.size(); i++) { ... }
```
> 初学阶段不致命，但养成好习惯可以避免以后踩坑。

## 🔍 调试小技巧

### 1. 注释排查法
把代码一行行注释掉，找到出问题的那一行：
```cpp
// cout << "测试1" << endl;  // 先注释掉
cout << "测试2" << endl;      // 如果能运行，问题在上一行
```

### 2. 输出调试法
在关键位置加 `cout`，看程序执行到哪一步：
```cpp
cout << "到这里了" << endl;    // 如果这行不输出，说明前面就崩了
int x = 10 / 0;               // 崩在这！
cout << "到这里了" << endl;    // 不会执行到
```

### 3. 检查符号
| ✅ 正确符号 | ❌ 错误符号 |
|-----------|-----------|
| `" "` 英文双引号 | `“ ”` 中文双引号 |
| `' '` 英文单引号 | `‘ ’` 中文单引号 |
| `;` 英文分号 | `；` 中文分号 |
| `()` 英文括号 | `（）` 中文括号 |

> 🌟 **黄金法则**：编译报错时，先看**第一个错误**！后面的错误往往是第一个引起的连锁反应。
